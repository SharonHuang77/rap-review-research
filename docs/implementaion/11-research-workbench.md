# 11 — Research Workbench

**Module:** Research Workbench

**Status:** Ready for Implementation

**Dependencies:**

- RFC-07 Research Evaluation Engine
- RFC-08 Hierarchical Review Architecture
- RFC-09 Consensus Review Architecture
- RFC-10 Export Service

---

# 1. Purpose

The Research Workbench provides an interactive interface for exploring experiment results.

Unlike a production dashboard, it is designed specifically for researchers.

The Workbench enables:

- browsing experiments
- comparing review architectures
- replaying multi-agent conversations
- inspecting evaluation metrics
- exporting datasets
- preparing research figures

The Workbench does **not** execute experiments.

It visualizes artifacts that have already been generated by the research platform.

---

# 2. Research Motivation

The research platform now supports three review architectures:

- Agentless
- Hierarchical
- Consensus

Each experiment produces:

- experiment metadata
- validated review findings
- evaluation metrics
- conversation history
- consensus votes
- export datasets

The Research Workbench provides a unified interface for exploring these artifacts.

---

# 3. Responsibilities

The Workbench is responsible for:

- browsing experiments
- comparing architectures
- replaying conversations
- visualizing evaluation metrics
- visualizing review findings
- viewing export history
- launching dataset exports

The Workbench is **not** responsible for:

- running experiments
- computing evaluation metrics
- calling LLM providers
- writing CSV files
- modifying stored artifacts

---

# 5. Design Principles

The Workbench follows these principles:

- read-only interface
- reproducible visualization
- deterministic rendering
- reusable services
- clear separation of UI and backend
- architecture comparison first

# 9. Workbench Service

```ts

    getComparison(snapshotId: string);
```

The Workbench aggregates existing backend services.

# Summary

The Research Workbench completes the research platform by providing a dedicated researcher-facing interface for exploring, comparing, replaying, and exporting experimental results.

Unlike a production dashboard, the Workbench is designed to support scientific experimentation and reproducibility. It separates visualization from experiment execution, consumes existing backend services through a clean API layer, and enables researchers to inspect experiment artifacts, compare communication topologies, replay multi-agent interactions, and export datasets for downstream analysis without introducing business logic into the presentation layer.
